로그인/로그아웃 UI (react-icons 활용)
메모 CRUD 기능 구현

src/components/MemoCard.js - 카드 형태 메모 UI (수정/삭제 버튼 포함)

React Query로 Supabase 데이터 패칭 및 캐싱 (useQuery, useMutation)

라우팅 및 스타일링

src/App.js에서 react-router-dom 설정 (/, /login, /signup)
src/styles/MemoCard.css, MemoList.css로 카드 레이아웃 스타일링
vibe05 패턴 참고: CSS 변수로 다크/라이트 테마 지원 (선택사항)

Vercel 배포

Supabase 무료 티어 제한 - 월 500MB DB, 2GB 전송량, 50,000 MAU 확인 필요. 초과 시 과금 또는 제한 발생

에러 처리 전략 - Supabase 에러 (네트워크, 인증 만료, RLS 거부) 대응을 위한 ErrorBoundary 또는 토스트 알림 추가?

테스트 흐름

회원가입: 프론트에서 이메일/사용자명/비밀번호로 회원가입 → 토큰 발급

자주 묻는 문제 해결

500 또는 Prisma 오류:
백엔드 폴더에서 npx prisma generate 다시 실행
npx prisma migrate dev로 초기 마이그레이션 재적용
CORS 또는 401:
프론트에서 로그인 후 요청에 Authorization: Bearer <token> 헤더 포함
포트 충돌:
백엔드 PORT 환경변수로 변경 가능, 프론트 proxy도 동일 포트로 유지
다음 단계 (Supabase/Postgres 마이그레이션)

Supabase에서 Postgres 준비 후, 다음을 진행합니다:
Prisma datasource를 Postgres URL로 변경 (예: postgresql://...)
npx prisma migrate deploy로 마이그레이션
서버의 Prisma 클라이언트 재생성(npx prisma generate) 및 재기동
원하시면 Supabase 테이블/정책(SQL)와 환경변수 세팅(.env)까지 제가 이어서 셋업해드릴게요.
